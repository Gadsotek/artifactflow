# Code of Conduct

ArtifactFlow contributors are expected to keep project spaces focused, respectful, and useful.

## Expected behavior

- Be direct, constructive, and considerate.
- Assume security-sensitive details may put users at risk and route them through private disclosure.
- Keep disagreement technical and specific.
- Respect maintainers' decisions about project scope, security boundaries, and release timing.

## Unacceptable behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private vulnerability details before a fix is available.
- Repeatedly derailing issues, pull requests, or discussions after maintainers ask you to stop.
- Sharing private contact details, credentials, logs, or content without permission.

## Scope

This code of conduct applies in all project spaces — the repository, issues, pull requests, discussions, and any project communication channel — and when an individual is representing the project in public spaces.

## Enforcement

Maintainers may edit, hide, or delete comments; close issues or pull requests; block accounts; or report abuse to the hosting platform when needed.

## Reporting

Report conduct concerns privately to `gadsotek@gmail.com`. Reports are handled confidentially; please include links and context so they can be acted on. This is a solo-maintained project: the aim is to acknowledge a report within a few days and to keep the reporter informed of the outcome. Maintainers will not retaliate against anyone who reports in good faith.
