The load-bearing decision

Run the code. Keep the boundary.

ArtifactFlow treats generated HTML as hostile executable content. It does not try to scrub arbitrary JavaScript until it becomes trustworthy. It keeps execution somewhere your authenticated application is not.

01 · App origin

The place that knows who you are.

  • Login and authenticated session cookie
  • Workspace and page authorization
  • Search, editing, versions, audit
  • Never executes artifact HTML
saved preview → short-lived
signed URL
02 · Artifact origin

The place that assumes the code is hostile.

  • No application cookies
  • Opaque sandboxed iframe
  • CSP blocks network, forms, frames, workers
  • Inline scripts may run; isolation does the work
Honest boundary: Secret scanning and the injected JavaScript guard are defense in depth. They are bypassable. The browser-enforced origin, iframe sandbox, and response-header CSP are the security boundary.

Humans and agents, same library

Agents put work where the team can find it.

ArtifactFlow does not generate content. Its MCP server lets approved AI clients search, read, create, update, and revert pages through the same application rules as human users.

Tokens can be read-only or read-write and scoped to selected workspaces.
MCP authority is hard-capped at Editor; administrator power is stripped in code.
Page content is returned inside an explicit untrusted-data envelope.
Writes keep scanning, versioning, concurrency checks, events, and audit attribution.

Your infrastructure

Easy to try. Serious about production.

The repository ships a local Compose stack and a hardened production image. It does not pretend one development file can be a safe universal production deployment.

Local evaluation · five commands
$ git clone https://github.com/Gadsotek/artifactflow

Production self-hosting

One image. Separate responsibilities.

app artifact-host worker scheduler

Bring two HTTPS hostnames, PostgreSQL with verified TLS, private persistent storage, secrets, mail, and correctly scoped proxy wiring. Set APP_ENV=production; the boot gate refuses an incomplete security contract.

Public alpha · July 2026

Built, tested, and still young.

Pin a revision. Expect breaking changes. Read the threat model before trusting it with work that matters.

  • Single-file HTML only; network access from artifacts is blocked by design.
  • No public sharing, marketplace, approval workflow, or built-in AI generation.
  • Version restore exists; a visual diff UI does not yet.
  • Automatic orphan-file garbage collection is deferred; operators have a report-first manual command.
  • Internally adversarially reviewed and broadly tested, but not independently audited by a third party.
